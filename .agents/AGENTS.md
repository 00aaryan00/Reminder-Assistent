# Workspace Rules

## Interaction and Execution Workflow

1. **No Big Leaps:** Do not implement large chunks of functionality all at once. Break tasks down into very small, manageable steps.
2. **Teach the Thought Process:** Before writing any code, explicitly explain the thought process, the architecture, and how the concepts apply to the current task. 
3. **Mandatory Approval:** Always present the proposed plan or steps and wait for explicit user approval *before* applying any code changes. Do not execute code automatically without discussion.
4. **Detailed Post-Change Explanation:** After making changes to the codebase, explain every part of the newly added or modified code in detail to ensure complete user understanding.
